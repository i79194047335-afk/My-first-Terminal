# Фаза 3 — Lighter как источник данных

Ветка: `phase-3-lighter`. Влить в `master` **только после полного завершения**.

Все решения ниже согласованы с владельцем 2026-07-18. Цифры — измеренные живым
прогоном, не оценки.

## Область

валюту (`retention.json` → `markets.fxcm`), Lighter — крипту, металлы, нефть,
индекс, акцию. Оба фида работают одновременно, хаб не различает источники.

## Хранение

| Поток | Хранение | Назначение |
|-------|----------|------------|
| Свечи + vol_base/vol_quote + дельта | 2000 баров/ТФ, вечно | график |
| Профиль объёма | корзины 0.05%, вечно | уровни |
| Тики сырьём | **кольцо 14 дней** | рэндж-бары |
| Лента крупных сделок | не хранится | живой поток |

Тики — кольцевой буфер: нужны только для рэндж-баров (из свечей внутрибарный
путь не восстановить). Ротация посуточная. **Необратимо:** глубже 14 дней
рэндж-бар другого размера не построить. Владелец принял осознанно.

Профиль объёма хранится **готовыми корзинами**, а не пересчитывается из тиков —
иначе умрёт вместе с ротацией. Корзина ~сотня чисел на день/инструмент.

Оценка диска: ~2.5-3 ГБ steady-state при 22 ГБ свободных.

## Проверенные факты API (живой прогон, не перепроверять)

Из ROADMAP (2026-07-12) — канал `trade/{id}`:
- `resolution` только `1m 5m 15m 1h 4h 1d`; голые числа → 400.
- `t` — миллисекунды. WS отдаёт `price`/`size` строками, REST — числами.
- `v` = base, `V` = quote. Хранить оба.
- `mark_price`/`index_price` нет в Pydantic-моделях SDK → raw REST.
- `count_back` непредсказуем → только `start`/`end` + пагинация по 500.
- Сервер шлёт `{"type":"ping"}` → обязателен `{"type":"pong"}`, иначе разрыв ~2 мин.

Добавлено этой фазой (прогон 2026-07-18):
- REST `/api/v1/orderBooks` → 226 рынков; `/api/v1/orderBookDetails` → 218 с
  объёмами (`daily_quote_token_volume`, `daily_trades_count`).
- `subscribed/trade` присылает снапшот последних ~50 сделок — **пропускать**,
  они продублируются через `update/trade` (иначе дубли на переподключении).

## Слои

Каждый слой проверяется изолированно до перехода к следующему.

### Слой 1 — фид
`feeds/lighter_feed.py` (py3.10) + `feeds/lighter_raw.py` (общий модуль).

**Модуль кладётся в `feeds/`, а НЕ в `core/`.** `core/` обязан парситься на
py3.7 (там же бежит FXCM-фид), а код бота — чистый py3.10: аннотации
`dict | None` (PEP 604) и `list[int]` (PEP 585) на 3.7 дают SyntaxError.
`feeds/lighter_raw.py` импортирует только py3.10-процесс. Файлы `core/*`
слой 1 не трогает вовсе.

- `_normalize_trade`, `Deduper`, ping/pong — портировать из бота
  (`/root/projects/Litr_v_odnogo/src/collector/lighter_ticks.py`, 8 суток аптайма).
- Разобрано живьём (2026-07-18), переносить как есть:
  * `_normalize_trade` → `{"m","p","s","t"(мс),"side"("buy"/"sell"),"tid"}`;
    `None` при отсутствии полей, при `is_maker_ask` не-`bool`, при ошибке
    приведения типов. Сторона агрессора уже готова — годится для дельты (слой 2)
    и разделения профиля buy/sell (слой 4).
  * `Deduper` — `set` + `deque`, окно 50 000 `tid`/рынок (≈час по BTC).
  * Реконнект — `tenacity`: экспоненциальный backoff 1→60 с, бесконечно;
    при реконнекте состояние обнуляется, снапшот `subscribed/trade`
    пропускается (размен: возможна потеря сделок в момент разрыва ради
    гарантии отсутствия дублей).
  * Зависимости бота: `websockets`, `tenacity`, `loguru`, `PyYAML`.
    В терминале логирование — свой `core/logfmt`, `loguru`/`yaml` не тянуть.
- Конвертация мс→сек **на границе фида** (`validate()` шины ловит мс как ошибку).
- Тики → шина; сырьё → `data/` с посуточной ротацией.
- Бэкфил при старте: REST `/api/v1/candles`, 2000 баров M1 = 4 запроса по 500.
  S5–S30 бэкфила не имеют (минимум API = `1m`), копятся живьём.
- Свой systemd-юнит `chart-feed-lighter`, отдельный venv (py3.10).

### Слой 2 — объём и дельта в свече
- `CandleBuilder.ingest(price, ts)` → принять `size` и сторону агрессора.
  **Обратная совместимость с FXCM обязательна** (`size=None`).
- Копить `vol_base`, `vol_quote`, `delta` (агрессивные покупки − продажи;
  сторона из `is_maker_ask`).
- Колонки `vol_base`/`vol_quote` в `candles` **уже есть** (Фаза 2.3);
  под дельту нужна миграция.

### Слой 3 — ротация тиков
Кольцевой буфер 14 дней для `provider=lighter`. Тики FXCM — вечный архив,
**не трогать** (CLAUDE.md).

### Слой 4 — профиль объёма
Корзины 0.05% от цены (параметр на инструмент), привязка к фиксированной сетке
от опорной цены — иначе профили разных дней не сложатся при движении цены.
Разделение buy/sell по стороне агрессора. Хранение вечное.

**Учесть выходные (замер 2026-07-18/19).** Синтетика на традиционные рынки
живёт по расписанию базового актива — доля минуток со сделками:

| Инструмент | Пятница | Суббота |
|------------|---------|---------|
| XAG        | 76%     | 19%     |
| MU         | 87%     | 28%     |
| SPCX       | 75%     | 45%     |
| XAU        | 94%     | 58%     |
| WTI        | 99%     | 100%    |
| BTC        | 100%    | 100%    |

Профиль по XAG/MU/SPCX/XAU нельзя считать по календарным суткам: выходные
размажут картину и дадут ложные «уровни» там, где просто никто не торговал.
Нужен либо профиль по торговым сессиям, либо явная пометка неполноценного бара.
Пустые минутки биржа отдаёт свечами с нулевым объёмом, а не пропускает.

### Слой 5 — лента крупных сделок
Порог — 95-й процентиль за скользящий час + абсолютный минимум (страховка от
вырожденного случая; по умолчанию можно 0). Пока окно не набрано — **молчать**,
а не показывать мусор. Не хранится.

**Минимум сделок в окне обязателен:** в выходные по XAG/MU скользящий час
наберёт единицы сделок, и процентиль по ним объявит «крупной» случайную мелочь.

### Замечание об отборе инструментов
Обороты, по которым отбирались 12 рынков, снимались в СУББОТУ — в будни они
выше. Список скорее недооценён, чем переоценён; пересматривать не нужно.

### Слой 6 — фронт: префикс символов
`fxcm:EUR/USD` / `lighter:BTC`. Отложено с Фазы 2.3, дальше тянуть нельзя:
16 инструментов от двух провайдеров. Затрагивает `currentSymbol` (~22 места
в `index.html`) + localStorage + миграция сохранённого состояния.
Гистограмма объёма отдельной паной (LWC `HistogramSeries`), скрыта при
`has_volume = false` (FXCM).

## Правила

- `chart-hub` / `chart-feed` **не рестартить** без явного разрешения.
- Ветка не вливается в `master` до полного завершения фазы.
- Docstrings ко всем функциям. `core/` — синтаксис, совместимый с py3.7
  (там же бежит FXCM-фид), новый lighter-фид — py3.10.
